HỆ THỐNG ĐẶT MÓN QUA QR CODE CHO NHÀ HÀNG / QUÁN ĂN / QUÁN CÀ PHÊ

Phiên bản cập nhật: 2026-04-01

1. Mục tiêu hiện tại của dự án

Dự án đang được phát triển theo hướng web app QR ordering dùng cho 3 nhóm người dùng:
- Khách hàng: quét QR, xem menu, chọn món, đặt nhiều lần trong cùng phiên bàn, yêu cầu thanh toán.
- Nhân viên: theo dõi order mới, xem trạng thái bàn, xử lý order, in phiếu bếp và hỗ trợ thanh toán.
- Chủ quán/Admin: quản lý menu, danh mục, bàn, QR, nhân viên và theo dõi dashboard vận hành.

Mục tiêu ngắn hạn của codebase hiện tại là hoàn thiện MVP có thể chạy nội bộ:
- Frontend tách riêng bằng Next.js.
- Backend tách riêng bằng Node.js.
- Dữ liệu được lưu qua lớp database cục bộ thay vì chỉ phụ thuộc mock API.
- Tăng độ an toàn cho khu staff/owner bằng JWT và route guard phía server.

2. Hướng kiến trúc hiện tại

Frontend:
- Next.js App Router trong thư mục `Frontend`
- Vai trò public/customer tại `/customer`
- Vai trò staff tại `/staff`
- Vai trò owner tại `/owner` và `/owner/qr-codes`
- Frontend gọi backend độc lập qua `NEXT_PUBLIC_API_BASE_URL`

Backend:
- HTTP server Node.js trong thư mục `Backend`
- Route nghiệp vụ tập trung ở `Backend/src/routes/index.js`
- Service xử lý nghiệp vụ ở `Backend/src/services/business.js`
- Dữ liệu được đồng bộ qua SQLite cục bộ tại `Backend/data/store.db`
- Vẫn giữ JSON snapshot `Backend/data/store.json` để seed và dễ kiểm tra dữ liệu

3. Trạng thái triển khai sau khi cập nhật

Các hạng mục đã được đưa vào code:
- Kết nối database cục bộ bằng SQLite qua `node:sqlite`
- Tự seed dữ liệu demo nếu chưa có dữ liệu
- JWT cho đăng nhập staff/owner
- Authorization guard cho các API staff/owner
- Frontend staff/owner gửi Bearer token khi gọi API
- Frontend customer hỗ trợ chọn option/topping của món khi menu có cấu hình
- Owner có thể cấu hình category, image URL, option/topping cho món ăn ngay trên giao diện quản trị

4. Hướng đi tiếp theo nên ưu tiên

Ưu tiên 1:
- Chuẩn hóa schema cuối cùng giữa tài liệu, SQL migration và backend runtime
- Chốt rõ chọn SQLite tạm thời hay chuyển hẳn sang PostgreSQL production

Ưu tiên 2:
- Bổ sung realtime bằng WebSocket/Socket.IO cho staff và bếp
- Có âm thanh/thông báo order mới

Ưu tiên 3:
- Hoàn thiện luồng thanh toán production:
  - VietQR động chuẩn
  - IPN/callback
  - đối soát giao dịch

Ưu tiên 4:
- Hoàn thiện auth production:
  - refresh token hoặc secure cookie
  - session management chặt hơn
  - phân quyền sâu hơn theo từng hành động

5. Đánh giá hiện trạng dự án

Dự án hiện tại đã đi đúng hướng MVP:
- Có tách frontend/backend
- Có luồng khách gọi món không cần đăng nhập
- Có dashboard staff và owner
- Có quản lý menu, category, staff, QR bàn
- Có nền tảng database cục bộ thật để tiếp tục mở rộng

Nhưng chưa phải production-ready vì còn thiếu:
- realtime thật
- callback thanh toán thật
- export báo cáo
- xác thực phiên an toàn hơn localStorage ở phía trình duyệt
- migration/schema chuẩn hóa hoàn toàn với tài liệu học thuật ban đầu

6. Kết luận ngắn

Hướng đi hiện tại của dự án là giữ phạm vi MVP thực dụng, làm chắc 3 lớp:
- FE sử dụng được trong quán
- BE đủ rõ ràng để mở rộng
- DB đủ ổn định để bỏ dần mock/demo

Sau bản cập nhật này, dự án đã tiến thêm một bước quan trọng: từ mô hình demo dùng store file đơn giản sang kiến trúc có database local, auth token và phân quyền API rõ hơn, phù hợp để tiếp tục triển khai realtime và thanh toán thật ở các vòng sau.
