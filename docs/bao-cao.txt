BAO CAO DINH HUONG DU AN
HE THONG DAT MON QUA QR CODE CHO NHA HANG / QUAN AN / QUAN CAFE

Phien ban cap nhat: 2026-04-01

1. Muc tieu hien tai cua du an

Du an dang di theo huong web app QR ordering co the chay local theo mo hinh frontend va backend tach rieng, trong do:
- Khach quet QR, xem menu, dat nhieu lan trong cung session ban va gui yeu cau thanh toan.
- Nhan vien theo doi order moi gan realtime, xu ly trang thai order va thanh toan.
- Chu quan quan ly menu, category, ban, QR, nhan vien, voucher va dashboard tong hop.

Muc tieu ngan han cua codebase hien tai:
- Giu frontend Next.js va backend Node.js doc lap.
- Chuyen nen du lieu sang PostgreSQL lam datastore chinh.
- Chot mot schema duy nhat dung chung cho docs, SQL va backend runtime.
- Dung cookie-first auth cho staff/owner voi access token + refresh token + session luu phia server.

2. Huong kien truc hien tai

Frontend:
- Next.js App Router trong thu muc Frontend
- Customer tai /customer
- Staff tai /staff
- Owner tai /owner va /owner/qr-codes
- Frontend goi backend doc lap qua NEXT_PUBLIC_API_BASE_URL

Backend:
- HTTP server Node.js trong thu muc Backend
- Route nghiep vu tap trung o Backend/src/routes/index.js
- Service nghiep vu o Backend/src/services/business.js
- Data store runtime dong bo vao PostgreSQL
- Co the xuat JSON snapshot de debug, nhung khong con dung JSON/SQLite lam datastore chinh

Database PostgreSQL:
- Schema chot tai Frontend/scripts/01-init-schema.sql
- Seed mau khop schema tai Frontend/scripts/02-sample-data.sql
- Co bang auth_sessions de quan ly refresh token/session server-side

3. Nhung huong da duoc cua ban cap nhat nay

Da bo sung va nang cap:
- Chuyen backend sang PostgreSQL thay cho SQLite/JSON lam nguon du lieu chinh
- Chuan hoa schema theo cac bang:
  - users
  - restaurants
  - tables
  - categories
  - menu_items
  - vouchers
  - sessions
  - orders
  - order_items
  - payments
  - activity_logs
  - auth_sessions
- Chuan hoa field nguoi dung ve full_name va dong bo name trong runtime
- Chuan hoa cac status quan trong:
  - order: pending, acknowledged, sent_to_kitchen, preparing, completed, cancelled
  - payment: pending, success, failed, refunded
  - table: empty, occupied, payment_requested, payment_pending, out_of_service
  - session: active, payment_requested, payment_pending, paid
- Auth client va server chuyen sang huong cookie-first dung nghia:
  - access token cookie httpOnly
  - refresh token cookie httpOnly
  - session phia server trong auth_sessions
  - endpoint session va refresh rieng
- Loai bo phu thuoc runtime vao Supabase Auth va localStorage lam session chinh
- Seed data da khop schema PostgreSQL va khop backend runtime hien tai

4. Dinh huong uu tien tiep theo

Uu tien 1:
- Hoan thien WebSocket / Socket.IO hai chieu thay cho SSE MVP

Uu tien 2:
- Tiep tuc day du CRUD categories va staff theo muc production hon:
  - validation
  - pagination / filter
  - audit log day du

Uu tien 3:
- Payment production:
  - tich hop that su voi MoMo Business API / ngan hang
  - mapping provider transaction id
  - dashboard doi soat day du theo ky

Uu tien 4:
- Hardening:
  - validation
  - migration strategy bai ban hon
  - test integration
  - backup / monitoring

5. Danh gia hien trang du an

Codebase hien tai da dung huong MVP mo rong:
- Frontend/backend tach rieng
- PostgreSQL la datastore chinh
- Session ban, order, payment, voucher va analytics da co data model ro rang
- Auth staff/owner da bo localStorage lam session chinh
- Payment callback/IPN demo da co

Nhung van chua production-ready vi con thieu:
- WebSocket / Socket.IO
- payment gateway that su voi credentials production
- validation/test tu dong bai ban
- chien luoc multi-tenant va sao luu du lieu

6. Tien do phase 0 va phase 1

Phase 0:
- Da chot lai schema du lieu duy nhat
- Da sua sample data va convention status
- Da bo hardcode Supabase/Auth cu
- Da co env mau cho backend va frontend

Phase 1:
- Backend doc lap da chay theo huong PostgreSQL
- Co migration schema PostgreSQL va seed
- Co auth JWT + refresh token + session server-side cho staff/owner
- Co public API cho customer load menu, QR, session, order

7. Ket luan ngan

Huong di hien tai cua du an da chuyen ro sang mot MVP co nen du lieu va auth dung hon de co the tiep tuc mo rong. Viec chot schema PostgreSQL, seed that su, va bo localStorage/Supabase khoi session chinh la buoc quan trong de dua codebase ra khoi trang thai demo thu cong va sang muc co the van hanh local nghiem tuc hon.
